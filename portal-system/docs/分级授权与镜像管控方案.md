## 一、授权码新格式（v2）

**格式：** `LS-{套餐码}{周期码}-{时间哈希}-{随机段}-{校验位}`

| 段 | 说明 | 示例 |
|----|------|------|
| 前缀 | 固定 `LS` | LS |
| 套餐码 | B/P/T/E/D | P = 专业版 |
| 周期码 | M/Q/Y/I | Y = 年付 |
| 时间哈希 | 4 位 | A3K9 |
| 随机段 | 6 位（排除 O/0/I/1/L） | X7M2R4 |
| 校验位 | 2 位 CRC | Q8 |

**示例：** `LS-PY-A3K9-X7M2R4-Q8`（专业版 年付）

**兼容：** 旧格式 `LINSCIO-XXXX-XXXX-XXXX-XXXX` 仍可激活（按 `plan_type` 处理）；新生成统一使用 LS- 格式。

## 三、镜像 Tag 与可拉取范围

| 套餐 | 可拉取镜像 Tag |
|------|----------------|
| 基础版 | `linscio-db:*`、`linscio-ai:basic*` |
| 专业版 | 上述 + `linscio-ai:pro*` |
| 团队版 | 上述 + `linscio-ai:team*` |
| 旗舰版 / 内测版 | 全部 |

激活时写入 `registry_credentials.allowed_image_tags`，供前端展示与后续 Registry 拉取校验使用。

## 四、Registry 用户名

- **新格式：** `{plan_prefix}_user_{uuid8}`，如 `basic_user_a1b2c3d4`、`pro_user_f5e6d7c8`。
- **plan_prefix：** basic / pro / team / enterprise / beta（与套餐 code 一致，便于识别与管控）。

## 五、数据库扩展（portal-system）

在现有表基础上新增或扩展：

- **module_definitions**：受控模块与套餐开放规则（bit 位、basic_enabled、pro_enabled、team_enabled）。
- **plans**：套餐定义（code、plan_char、machine_limit、concurrent_limit、pull_limit_monthly、price_monthly）。
- **billing_periods**：付费周期（code、period_char、months、discount_rate）。
- **license_keys**：增加 plan_id、period_id、module_mask、machine_limit、concurrent_limit、pull_limit_monthly；保留 plan_type 兼容旧数据。
- **registry_credentials**：增加 license_id、allowed_image_tags (TEXT[])。
- **machine_bindings**、**instance_heartbeats**、**pull_records**：供主应用或 Registry 鉴权使用，可在 portal-system 中建表并预留 API。

详见 `docs/数据库设计.md` 更新后的 9.x 节。

## 六、激活流程（portal-system）

1. **① 格式校验**：LS- 格式 CRC 校验，失败→「授权码格式错误，请检查输入」。**② 解析套餐/周期**：LS- 码须能解析且套餐/周期在库中存在，否则→「授权码无效」。**③ 数据库查询**：不存在/已使用/已作废/已过期 分别返回对应文案。**④ 生成凭证**：`registry_username = {plan_prefix}_user_{uuid8}`，`registry_password = token_hex(12)`，htpasswd + AES + allowed_image_tags。**⑤⑥** 标记已使用、返回成功。**8.3 运行时心跳**：`POST /api/v1/license/heartbeat`（license_key, machine_id, instance_id）；授权已过期→200 且 license_expired 为 true；机器绑定满且本机未绑或并发超限→403；超过 15 分钟无心跳标记实例离线。

## 七、管理后台（portal-system）

- **批量生成授权码**：支持选择套餐（basic/professional/team/enterprise/beta）、周期（monthly/quarterly/yearly）、数量、备注；生成 LS- 格式授权码，并写入 plan_id、period_id、module_mask、machine_limit、concurrent_limit、pull_limit_monthly。
- **授权码列表/导出**：展示套餐、周期、机器数/并发数/月拉取上限、绑定用户、状态（未使用/已激活/已过期/已作废）。
- **模块权限配置**：`GET /admin/modules` 列表（可选 controlled_only）；`PATCH /admin/modules/{module_id}` 更新 basic_enabled、pro_enabled、team_enabled、sort_order；`POST /admin/modules` 新增受控模块（自动分配 bit）；修改后新生成授权码按最新配置计算 module_mask。
- **批量更新凭证**：`POST /admin/registry-credentials/sync-modules` 按当前套餐模块配置刷新所有已激活用户的 allowed_image_tags，使新模块无感知生效。
- **授权码列表/详情**：`GET /admin/licenses` 返回套餐名、周期名、状态、绑定用户名等；`GET /admin/licenses/{id}` 查看详情（含机器绑定数、并发实例数、本月拉取次数）。
- **导出 CSV**：`GET /admin/licenses/export` 字段含授权码、套餐、周期、机器数、并发数、月拉取上限、有效期至、生成时间、备注等。
- **授权码延期**：`PATCH /admin/licenses/{license_id}/extend`，请求体 `{ "months": 1|3|6|12 }`，同步更新对应 registry_credentials.expires_at。
- **数据总览**：`GET /admin/stats` 含各套餐存量分布、本月新增激活数、7 天内到期预警、各套餐活跃用户数、机器绑定总数、本月拉取 Top 10、异常监控（拉取剩余≤3、同机多账号等）。

## 八、用户中心（门户前端）

- 展示当前套餐名称、周期、有效期、剩余天数。
- 展示本套餐可拉取镜像 Tag 示例（来自 `allowed_image_tags`）。
- 拉取命令中镜像名按套餐 Tag 展示（如 `linscio-ai:basic`、`linscio-ai:pro`）。
- **机器管理**：机器指纹参考 `scripts/machine_fingerprint.py`；用户中心机器列表含套餐名、剩余槽位、在线状态、自助解绑说明；管理员解绑接口对基础/专业版校验每周期更换次数。
