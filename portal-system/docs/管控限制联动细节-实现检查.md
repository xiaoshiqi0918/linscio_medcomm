# 管控限制联动细节 — 实现检查

对照 5.1～5.6 的联动路径，核对实现与文档的一致性。路径以《三系统职责边界》为准。

---

## 5.1 作废授权码 → 三端联动

| 步骤 | 文档描述 | 实现 |
|------|----------|------|
| 数据库 | is_revoked=TRUE, credentials.expires_at=NOW() | ✅ revoke_license 已设置并写库 |
| htpasswd | 删除该用户凭证行 | ✅ remove_from_htpasswd(registry_username) |
| 门户 | 下次请求返回「已作废」 | ✅ GET /user/license 仅返回未作废授权，已作废视为未激活 |
| Registry | 凭证失效，无法 docker login/pull | ✅ htpasswd 已删；proxy/token 校验 is_revoked |
| 应用实例 | 5 分钟内心跳返回 403，显示「授权已失效」 | ✅ 已作废授权心跳返回 403，detail「授权已作废」 |

## 5.3 模块权限变更 → 三端联动

| 步骤 | 文档描述 | 实现 |
|------|----------|------|
| 数据库 | module_definitions 更新 | ✅ 管理端 GET/PATCH/POST /admin/modules |
| 新授权码 | module_mask 自动包含新模块 | ✅ create_license_batch_v2 写入 module_mask |
| 批量更新凭证 | 已激活用户手动触发 | ✅ POST /admin/registry-credentials/sync-modules |
| 单用户更新凭证 | 用户管理页对单用户刷新凭证（按当前套餐重算权限与密码） | ✅ POST /admin/users/{user_id}/refresh-credential |
| 凭证更新内容 | allowed_image_tags + license_keys.module_mask | ✅ sync-modules 更新 cred.allowed_image_tags 与 lk.module_mask |
| 门户 | 模块入口从锁定变为可用 | ✅ 依赖 license 与模块配置展示 |
| Registry | allowed_image_tags 更新，可拉取更高级镜像 | ✅ sync-modules 刷新 allowed_image_tags |
| 应用实例 | 下次心跳返回新 module_mask，应用解锁模块入口 | ✅ 心跳响应增加 module_mask 字段 |

## 5.4 生成次数超限 → 应用硬挂起

| 步骤 | 文档描述 | 实现 |
|------|----------|------|
| 请求 | POST /api/v1/quota/check，body: license_key, machine_id, content_type | ✅ POST /api/v1/quota/check |
| 未超限 | 200 OK，允许创建，used_count+1 | ✅ check_and_use_quota 扣减后返回 200 |
| 已超限 | 403，应用禁用新建按钮、展示周期重置日期 | ✅ 403，detail 含 message、used、limit、next_reset_date |
| 门户 | 生成次数展示「已用完」 | ✅ GET /user/quota-summary |
| 管理后台 | 可查看超限详情、可手动重置 | ✅ GET /admin/licenses/{id}/quota-usage，POST /admin/quota/reset |

## 5.5 拉取次数超限 → Registry 拒绝

| 步骤 | 文档描述 | 实现 |
|------|----------|------|
| 请求 | docker pull → Registry 请求 Token 服务 | ✅ GET /api/v1/registry/token（或走 proxy） |
| 校验 | ① 凭证有效 ② pull_count_this_month < pull_limit_monthly | ✅ Token 接口与 proxy 均做自然月重置 + 上限校验 |
| 超限 | 拒绝签发 Token，返回 429 | ✅ Token 接口返回 429，errors[].message「本月拉取次数已达上限」 |
| proxy | 拉取 manifest 前同样校验，403 | ✅ _ensure_pull_quota_and_check_limit 返回 403 |
| 门户/管理端 | 拉取次数展示、可重置 | ✅ 门户 quota-summary；管理端可查看与重置（拉取次数为 cred 字段，暂无单独重置接口时可依赖自然月重置） |

---

## 5.6 机器绑定数满 → 新机器被拒

| 步骤 | 文档描述 | 实现 |
|------|----------|------|
| 请求 | POST /api/v1/license/heartbeat | ✅ POST /api/v1/license/heartbeat |
| 校验 | 本机未绑定且已绑定数 = machine_limit | ✅ 心跳内查 machine_bindings，超限返回 403 |
| 返回 | 403，应用显示「已达机器绑定上限，请在用户中心解绑旧机器」 | ✅ 403，detail 同文案 |
| 门户 | 机器管理页展示满员、引导解绑 | ✅ GET /user/machines |
| 管理后台 | 可代用户强制解绑 | ✅ DELETE /admin/licenses/{id}/machines/{binding_id} |

---

**结论**：5.1～5.6 联动均已实现；本次补全包括：心跳返回 `module_mask`、sync-modules 同步 `license_keys.module_mask`、Token 接口拉取超限返回 429。路径与《三系统职责边界》一致。
