# LinScio 门户系统 · 部署前检查（腾讯云香港 + 宝塔）

**目标服务器**：腾讯云香港，公网 IP **43.161.217.10**  
**部署方式**：宝塔面板 + Docker Compose  

请按下列清单逐项确认，确保部署顺利进行。

---

## 一、服务器与环境

| 检查项 | 要求 | 操作/说明 |
|--------|------|-----------|
| 操作系统 | Ubuntu 22.04 / 24.04 LTS | 建议 24.04，与 [部署-腾讯云轻量与宝塔.md](部署-腾讯云轻量与宝塔.md) 一致 |
| 内存 | ≥ 4 GB | 4 GB 可运行全部服务；若 2 GB 需关闭部分服务或加 swap |
| 磁盘 | ≥ 30 GB 可用 | 含系统、Docker 镜像、数据库与备份 |
| 宝塔面板 | 已安装并可访问 | 用于 Nginx 反向代理、SSL、计划任务 |
| SSH 登录 | 可正常登录 | 用于执行 docker / 脚本 |

| 检查项 | 要求 | 操作/说明 |

若使用域名（推荐），在域名服务商处添加 A 记录，解析到 **43.161.217.10**：

| 主机记录 | 记录类型 | 解析结果 | 用途 |
|----------|----------|----------|------|
| @ | A | 43.161.217.10 | 根域（可 301 跳转到 www） |
| www | A | 43.161.217.10 | 门户主站 |
| admin | A | 43.161.217.10 | 管理后台 |
| registry | A | 43.161.217.10 | 镜像仓库（docker login/pull） |

## 四、Docker 与目录

| 检查项 | 要求 | 操作/说明 |
|--------|------|-----------|
| Docker 已安装 | `docker --version`、`docker compose version` 正常 | 未安装：`curl -fsSL https://get.docker.com \| sh`，然后 `sudo usermod -aG docker $USER`，登出再登录 |
| Docker 镜像加速 | 香港/国内构建不超时 | 若 `docker pull` 超时，配置 `/etc/docker/daemon.json` 的 `registry-mirrors`，见 [部署-腾讯云轻量与宝塔.md](部署-腾讯云轻量与宝塔.md) §2.1 |
| 部署目录 | `/www/wwwroot/linscio` 存在且当前用户可写 | `sudo mkdir -p /www/wwwroot/linscio && sudo chown -R $USER:$USER /www/wwwroot/linscio` |
| 项目位置 | 代码在 `/www/wwwroot/linscio/portal-system` | 上传或 git clone 后，确保该目录下有 `docker-compose.yml`、`api/`、`portal/`、`admin/`、`registry/`、`scripts/` 等 |

## 六、环境变量 .env

| 检查项 | 要求 | 说明 |
|--------|------|------|
| 模板 | 已有 `.env`，且由 `.env.production.example` 复制并修改 | `cp .env.production.example .env && chmod 600 .env` |
| **POSTGRES_PASSWORD** | 强密码，勿为空 | 数据库密码，首次启动后勿随意修改（见 README 排障） |
| **JWT_SECRET_KEY** | 强随机字符串（建议 64 字符） | 如 `openssl rand -hex 32` |
| **AES_SECRET_KEY** | 32 字节 | 如 `openssl rand -base64 24 \| head -c 32` |
| **AES_IV** | 16 字节 | 如 `openssl rand -base64 16 \| head -c 16` |
| **REGISTRY_URL** | 镜像仓库对外地址 | 有域名填 `registry.你的域名`，暂用 IP 则 `43.161.217.10:5000`（若 5000 通过 Nginx 代理则仍建议用域名） |
| **CORS_ORIGINS** | JSON 数组，含门户与管理后台前端地址 | 生产示例：`["https://www.你的域名.com","https://admin.你的域名.com"]`；若用 IP：`["http://43.161.217.10:3000","http://43.161.217.10:3001"]` |
| **ENVIRONMENT** | 填 `production` | 生产环境标识 |

可选：`COS_SECRET_ID`、`COS_SECRET_KEY`（下载中心 COS）；`REGISTRY_JWT_SECRET`（与门户 JWT 隔离）。

## 七、首次启动前必做

| 步骤 | 命令/操作 | 说明 |
|------|------------|------|
| 1. 生成 htpasswd | `./scripts/create-registry-htpasswd.sh` 或 `./scripts/create-registry-htpasswd.sh 用户名 强密码` | 未生成则 Registry 容器会 panic 退出 |
| 2. 构建镜像 | `docker compose build` | 首次较慢，依赖镜像加速 |
| 3. 启动服务 | `docker compose up -d` | 等待约 30～60 秒 |
| 4. 健康检查 | `docker compose ps` 中 linscio-api 为 `Up (healthy)` | 可再 `curl -s http://127.0.0.1:8001/health` |
| 5. 创建管理员 | `./scripts/create-admin.sh 管理员用户名 管理员密码` | 同步到数据库与 Registry 认证 |
| 6. 重启 Registry | `docker compose restart linscio-registry` | 使管理员账号可 docker login / Registry UI 登录 |

## 八、宝塔 Nginx 反向代理（推荐）

| 检查项 | 要求 | 说明 |
|--------|------|------|
| 站点/反向代理 | 门户、管理后台、Registry、Hub（及可选 API） | 参考 `deploy/baota-nginx-linscio.conf`，将 `server_name` 改为你的域名，`proxy_pass` 指向 `http://127.0.0.1:3000`、`:3001`、`:5000`、`:5001`、`:8001` |
| SSL | 为各域名申请证书并开启「强制 HTTPS」 | 宝塔可一键 Let’s Encrypt |
| 端口 | 仅 80/443 对外，3000/3001/8001/5000/5001 仅 127.0.0.1 | 已在 docker-compose 中绑定 127.0.0.1，无需在安全组开放 |

## 十、常见问题预防

| 问题 | 预防/处理 |
|------|-----------|
| 构建时拉取镜像超时 | 配置 Docker 镜像加速后重试 |
| Registry 启动报错 "could not create registry" | 先执行 `./scripts/create-registry-htpasswd.sh`，检查 `registry/auth/htpasswd` 存在 |
| 502 Bad Gateway | 先确认 `docker compose ps` 中 linscio-api 为 healthy，再检查宝塔反向代理是否指向 127.0.0.1:8001（或对应端口） |
| 管理员登录 401 / 无法登录 | 执行 `./scripts/create-admin.sh` 后重启 Registry |
| 更新后用户数据丢失 | **禁止**使用 `docker compose down -v`，更新只用 `docker compose build && docker compose up -d` |
| 数据库密码与 .env 不一致 | 首次启动后勿改 POSTGRES_PASSWORD；若已改，需按 README 恢复或重建库 |
