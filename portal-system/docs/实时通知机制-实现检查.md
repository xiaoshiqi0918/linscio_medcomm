# 实时通知机制 — 实现检查

对照 7.1～7.3 的刷新策略与心跳响应内容，核对实现与文档的一致性。

---

## 7.1 门户网站状态刷新

文档：轮询/按需请求，进入对应页时请求一次，无定时轮询。

| 数据 | 文档策略 | 实现 |
|------|----------|------|
| 授权状态/有效期 | 进入用户中心时请求一次 | ✅ Dashboard onMounted 调用 `userApi.license()`、`registryCredentialFromRegistry()` |
| 生成次数剩余 | 进入用户中心时请求一次 | ✅ 进入「生成次数」页（Quota.vue）时请求 `quotaSummary()` + `license()` |
| 机器绑定列表 | 进入机器管理页时请求一次 | ✅ Machines.vue onMounted 请求 `license()` + `machines()` |
| 授权即将到期提醒 | 登录后检查一次 | ✅ 进入用户中心即展示 `days_remaining`，无单独「登录后弹窗」 |

门户未做定时轮询，与「用户主动操作时按需请求」一致。

---

## 7.2 管理后台数据刷新

文档：看板每 60 秒刷新，异常预警每 30 秒刷新。

| 文档描述 | 实现 |
|----------|------|
| 看板数据每 60 秒自动刷新 | ✅ Stats.vue 在 onMounted 后 `setInterval(fetchStats, 60 * 1000)`，onUnmounted 时 clearInterval |
| 异常预警每 30 秒刷新 | ⚠️ 异常预警与看板共用 GET /admin/stats，当前与看板一起每 60 秒刷新；若需 30 秒可改为 30s 间隔或单独接口 |

---

## 7.3 应用实例心跳

文档：每 5 分钟上报，服务端 15 分钟无心跳标记离线；响应含 valid、module_mask、expires_at、days_remaining、concurrent_count、quota。

| 项目 | 文档 | 实现 |
|------|------|------|
| 心跳间隔 | 每 5 分钟 | 客户端行为，服务端不强制 |
| 离线判定 | 超过 15 分钟无心跳标记离线 | ✅ HEARTBEAT_OFFLINE_MINUTES = 15，心跳内将超时实例置 is_active=False |
| valid | true/false | ✅ 使用字段 `ok`，含义一致 |
| module_mask | 63 | ✅ 已返回 |
| expires_at | "2027-03-01" | ✅ 已返回 isoformat |
| days_remaining | 364 | ✅ 已返回，由 expires_at 与当前时间计算 |
| concurrent_count | 1 | ✅ 已返回 |
| quota | schola/medcomm/qcc 的 used、limit、reset_date | ✅ 已返回，结构为 `quota: { schola: { used, limit, reset_date }, ... }`，按本机本周期计算 |

应用收到心跳后可：更新本地 module_mask、更新生成次数展示、在 valid/ok=false 或 license_expired 时展示错误/到期提示。

---

**结论**：7.1 门户按需请求已实现；7.2 看板 60 秒自动刷新已实现，异常预警随看板 60 秒刷新；7.3 心跳响应已补全 days_remaining 与 quota，与文档一致。
