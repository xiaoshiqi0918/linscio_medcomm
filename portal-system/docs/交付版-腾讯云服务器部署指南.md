# LinScio AI · 聆思恪 门户系统 · 交付版腾讯云服务器部署指南

本指南适用于 **纯净交付包**（无 node_modules、无 Python 虚拟环境、无 .env/htpasswd、无 .git），在腾讯云服务器上从零部署。

---

## 交付包说明

- **纯净版内容**：仅含源码与配置，不含已安装依赖与密钥。
- **已排除**：`node_modules`、`api/.venv`、`portal/dist`、`admin/dist`、`.env`、`registry/auth/htpasswd`、`.git`、`__pycache__` 等。
- **部署方式**：全程通过 **Docker Compose** 构建与运行，无需在服务器安装 Node.js 或 Python；前端在镜像构建时由容器内 `npm install` 与 `npm run build` 完成。

---

## 一、服务器与环境

### 1.1 推荐配置

| 项目 | 建议 |
|------|------|
| 云服务商 | 腾讯云（轻量应用服务器或云服务器 CVM） |
| 规格 | 4 vCPU · 4 GB 内存或以上 |
| 系统 | Ubuntu 22.04 / 24.04 LTS |
| 可选 | 宝塔面板（用于 Nginx 反向代理与 SSL） |

### 1.2 域名与解析（生产必做）

将以下子域名 A 记录解析到服务器公网 IP（示例 IP 请替换为实际）：

| 主机记录 | 记录类型 | 解析结果 |
|----------|----------|----------|
| @ | A | 你的公网IP |
| www | A | 你的公网IP |
| admin | A | 你的公网IP |
| registry | A | 你的公网IP |
| hub | A | 你的公网IP |

## 二、制作交付包（在开发机执行）

在 **portal-system** 目录下执行：

```bash
chmod +x scripts/make-delivery.sh
./scripts/make-delivery.sh
```

会在上级目录生成两个文件：
- `portal-system-delivery-YYYYMMDD.tar.gz`
- `portal-system-delivery-YYYYMMDD.zip`

**交付建议**：若部署方使用腾讯云/宝塔的**网页文件管理**上传，请交付 **.zip** 包，可在文件管理中右键解压；**.tar.gz 在网页文件管理中无法直接打开**，需通过 SSH 用命令解压。

## 三、服务器准备

### 3.1 安装 Docker 与 Docker Compose

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# 登出并重新登录 SSH 后执行：
docker compose version
```

**3.1.1 配置 Docker 镜像加速（国内服务器必做，避免构建超时）**

国内服务器直接访问 Docker Hub 常会超时（如 `dial tcp ... i/o timeout`），导致 `docker compose build` 时无法拉取 `node:20-alpine` 等基础镜像。请配置镜像加速后再构建。

**腾讯云**：使用腾讯云容器镜像服务提供的公共镜像（若拉取时报 500，请改用下方其他镜像）：

```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<'EOF'
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com"
  ]
}
EOF
```

若上述镜像拉取时报 **500 Internal Server Error**，可改用其他国内镜像（任选其一或配置多个备用）：

```bash
# 示例：改用 docker.1ms.run 或 轩辕镜像
sudo tee /etc/docker/daemon.json <<'EOF'
{
  "registry-mirrors": [
    "https://docker.1ms.run",
    "https://docker.xuanyuan.me"
  ]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker
```

**其他国内环境**（阿里云、华为云等）：可改用对应厂商的镜像地址，例如：

- 阿里云：`https://你的ID.mirror.aliyuncs.com`（需在容器镜像服务控制台获取）
- 华为云：`https://华为云SWR地址`  
- 通用：`https://docker.1ms.run` 等（以各厂商当前文档为准）

配置完成后执行 `docker pull node:20-alpine` 测试，能拉取成功再执行 `docker compose build`。

### 3.2 创建部署目录

```bash
sudo mkdir -p /www/wwwroot/linscio
sudo chown -R $USER:$USER /www/wwwroot/linscio
cd /www/wwwroot/linscio
```

### 3.3 上传并解压交付包

将交付包上传到 `/www/wwwroot/linscio/`，然后按以下两种方式之一解压。

**方式一：使用 .zip（推荐，适用于宝塔/腾讯云网页文件管理）**

- 上传 `portal-system-delivery-YYYYMMDD.zip` 到 `/www/wwwroot/linscio/`。
- 在宝塔「文件」或腾讯云「文件管理」中进入该目录，**右键该 .zip 文件 → 解压**。
- 解压后得到 **portal-system** 目录。

**方式二：使用 .tar.gz（需 SSH 终端）**

- 腾讯云/宝塔的网页文件管理**无法直接打开或解压 .tar.gz**，需用 SSH 登录服务器后执行：

```bash
cd /www/wwwroot/linscio
tar -xzf portal-system-delivery-*.tar.gz
```

- 解压后得到 **portal-system** 目录。

**后续步骤**（任一种方式解压后相同）：所有命令在 **portal-system** 目录下执行：

```bash
cd /www/wwwroot/linscio
cd portal-system
```

## 四、配置环境变量

```bash
cd /www/wwwroot/linscio/portal-system
cp .env.production.example .env
chmod 600 .env
```

编辑 `.env`，**必改**项：

| 变量 | 说明 |
|------|------|
| `POSTGRES_PASSWORD` | 数据库强密码 |
| `JWT_SECRET_KEY` | 建议 64 位随机字符串 |
| `AES_SECRET_KEY` | 32 字节密钥（用于 Registry 密码加密） |
| `AES_IV` | 16 字节 IV |
| `REGISTRY_URL` | 镜像仓库对外域名，如 `registry.yourdomain.com` |
| `CORS_ORIGINS` | 前端域名 JSON 数组，如 `["https://www.yourdomain.com","https://admin.yourdomain.com"]` |
| `ENVIRONMENT` | 填 `production` |
| `DATA_DIR` | 可选，如 `/www/wwwroot/linscio/data` |

生成随机密钥示例（可选）：

```bash
openssl rand -hex 32   # 用作 JWT_SECRET_KEY 等
openssl rand -base64 24 | head -c 32  # AES_SECRET_KEY
openssl rand -base64 16 | head -c 16  # AES_IV
```

## 五、首次启动

### 5.1 生成 Registry 认证文件（必做）

未生成前不要启动 Registry，否则会报错退出。

```bash
chmod +x scripts/create-registry-htpasswd.sh
./scripts/create-registry-htpasswd.sh
# 生产建议指定强密码：./scripts/create-registry-htpasswd.sh 管理员用户名 强密码
```

### 5.2 构建并启动所有服务

```bash
docker compose build
docker compose up -d
```

等待数据库健康后再进行下一步（约 10～30 秒）：

```bash
docker compose ps
# 确认 linscio-db 为 healthy
```

### 5.3 创建管理员（并同步到镜像仓库）

```bash
chmod +x scripts/create-admin.sh
./scripts/create-admin.sh
# 或指定账号：./scripts/create-admin.sh 你的用户名 '你的密码'
```

若 Registry 已先启动，需重启以加载新认证：

```bash
docker compose restart linscio-registry
```

### 5.4 套餐与周期（批量生成授权码前）

**第一优先级**：在管理后台「套餐与周期」页中配置（至少一个套餐、一个付费周期），即可在「授权码管理」→「批量生成」中使用。

- **新部署**：登录管理后台 → 左侧「套餐与周期」→ 新增套餐、新增周期，无需执行脚本。
- **已有库升级**：若从旧版本升级且尚未有 `billing_periods.name` 列，执行一次：  
  `docker compose exec -T linscio-db psql -U linscio_user -d linscio_portal < api/scripts/migrate_billing_period_name.sql`
- **可选预置**：若希望一次性写入多组套餐/周期/模块数据，可执行：  
  `docker compose exec linscio-api python scripts/seed_plans_periods_modules.py`  
  预置后仍可在「套餐与周期」中增删改。

## 六、Nginx 反向代理（腾讯云 + 宝塔）

### 6.1 使用宝塔

1. 在宝塔「网站」中为 **根域**、**www**、**admin**、**registry**、**hub** 分别添加站点（或使用「反向代理」），域名与解析一致。
2. 在站点「设置」→「配置文件」中，将项目内 `deploy/baota-nginx-linscio.conf` 中对应 **server** 块合并进去。
3. 将配置中的 `server_name`、`proxy_pass` 端口与你的域名、本机端口对应：
   - 门户 → `http://127.0.0.1:3000`
   - 管理后台 → `http://127.0.0.1:3001`
   - 镜像仓库 API（registry）→ `http://127.0.0.1:5000`
   - 镜像仓库 Web UI（hub）→ `http://127.0.0.1:5001`
4. 在宝塔中为各域名申请 SSL，并开启「强制 HTTPS」。

### 6.2 端口说明

- 仅需对公网开放 **80、443**（由 Nginx 监听）。
- 3000、3001、8000、5000、5001 仅绑定 127.0.0.1，由 Nginx 反向代理到对应端口。

## 七、访问与验证

| 用途 | 地址 |
|------|------|
| 门户主站 | https://www.你的域名 |
| 管理后台 | https://admin.你的域名 |
| 镜像仓库（docker login/pull） | https://registry.你的域名 |
| 镜像仓库 Web UI | https://hub.你的域名 |

- 健康检查：`curl -s http://127.0.0.1:8000/health` 应返回 `{"status":"ok"}`。
- 镜像仓库代理：`curl -s http://127.0.0.1:5000/health` 应返回 `{"status":"ok","service":"registry-proxy"}`。

## 八、日常运维要点

### 8.1 更新/重启（保留数据）

- **仅重启**：`docker compose restart` 或 `docker compose up -d`。
- **更新代码后**：重新上传文件后执行 `docker compose build && docker compose up -d`。
- **严禁**：`docker compose down -v` 会删除数据库卷，导致用户与授权码等数据全部丢失；除非全新部署且不需保留任何数据，否则不要使用 `-v`。

### 8.2 数据备份

```bash
mkdir -p /www/wwwroot/linscio/data/backups
./scripts/backup-db.sh
```

建议在宝塔「计划任务」中配置每日执行上述备份命令。

### 8.3 恢复管理员/镜像仓库认证

管理员或 htpasswd 丢失时，在项目根目录执行：

```bash
docker compose up -d linscio-db linscio-api
./scripts/create-admin.sh 你的用户名 '你的密码'
docker compose restart linscio-registry
```

修改已有管理员密码：`./scripts/create-admin.sh 用户名 新密码 update`。

### 8.4 常见问题

| 现象 | 处理 |
|------|------|
| Registry 启动报错 / panic | 先执行 `./scripts/create-registry-htpasswd.sh`，再 `docker compose up -d` |
| 管理后台/门户 502 | 查看 `docker compose logs linscio-api`，确认 API 正常；`curl http://127.0.0.1:8000/health` 检查 |
| 批量生成授权码报「套餐或周期不存在」 | 先在管理后台「套餐与周期」中添加至少一个套餐和一个周期；或执行 `docker compose exec linscio-api python scripts/seed_plans_periods_modules.py` 预置 |
| 数据总览/授权码列表 500（缺列） | 执行 `./scripts/run-migrate-license-keys.sh` 后 `docker compose restart linscio-api` |

更多排障与 Nginx 配置示例见同目录 **部署-腾讯云轻量与宝塔.md**。
