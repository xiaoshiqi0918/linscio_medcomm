# 管理后台操作说明 — 最终实现检查

依据「管理后台操作说明」10.1～10.5 及附录，对 portal-system 的接口与前端逐项核对。✅ 表示已实现，❌ 表示未实现或与说明不一致，⚠️ 表示部分实现。

---

## 10.1 模块权限配置

| 项 | 说明 | 状态 | 备注 |
|----|------|------|------|
| 模块权限配置页面 | 管理员实时调整受控模块套餐开放规则 | ✅ | 后端 `GET/PATCH/POST /admin/modules`，前端「模块权限配置」页（列表 + 编辑套餐 + 新增 + 一键初始化受控模块） |
| 新增受控模块 | 填写名称、模块标识，勾选各套餐，系统自动分配下一 Bit 位 | ✅ | `POST /admin/modules` 已实现，自动 `max(bit_position)+1` |
| 修改后对新授权码生效 | 新生成授权码按最新 module_definitions 计算 module_mask | ✅ | `create_license_batch_v2` 调用 `get_module_mask(plan_code)` |
| 批量更新凭证 | 已激活用户无感知获得新模块权限（刷新 allowed_image_tags、module_mask、密码） | ✅ | 后端 `POST /admin/registry-credentials/sync-modules`，前端「模块权限配置」页「批量更新凭证」按钮 |
| 用户管理-更新凭证 | 对单个用户刷新镜像仓库凭证（按当前套餐重算权限与密码） | ✅ | 后端 `POST /admin/users/{user_id}/refresh-credential`，前端「用户管理」每行「更新凭证」按钮 |

**已实现**：模块权限配置页（含一键初始化受控模块、批量更新凭证）；用户管理页提供单用户「更新凭证」。

## 10.3 授权码列表

| 项 | 说明 | 状态 | 备注 |
|----|------|------|------|
| 授权码 | 完整码 + 一键复制 | ⚠️ | 有完整码，**无一键复制按钮** |
| 套餐 | Basic/Pro/Team/Enterprise/Beta | ⚠️ | 列表返回 `plan_name`，前端显示为「版本」且用 `plan_type`，v2 码应优先显示 plan_name |
| 周期 | 月付/季付/年付 | ⚠️ | 后端已返回 `period_name`，**前端未展示** |
| 机器数 | 已绑定/上限（不限 ∞） | ❌ | 详情接口有 machine_bound_count；**列表接口未带**，前端未展示 |
| 并发数 | 当前活跃/上限（不限 ∞） | ❌ | 详情有 instance_active_count；**列表未带**，前端未展示 |
| 本月拉取 | 已用/上限（不限 ∞） | ❌ | 详情有 pull_used_this_month；**列表未带**，前端未展示 |
| 有效期至 | 日期 + 剩余天数 | ⚠️ | 有日期，**无剩余天数** |
| 绑定用户 | 用户名，未激活显示「—」 | ✅ | 有，显示「已绑定」或「—」 |
| 状态 | 🟡未使用/🟢已激活/🔴已过期/⚫已作废 | ⚠️ | 有状态 tag，**未用颜色/图标区分**，已过期未单独标出 |
| 操作 | 查看详情 / 延期 / 作废 | ⚠️ | 仅有「作废」，**无「查看详情」「延期」** |

**待办**：  
- 列表接口增加每条 `machine_bound_count`、`instance_active_count`、`pull_used_this_month`（或保持仅详情有，列表只显示「查看详情」）。  
- 前端：展示周期、机器数(已绑定/上限)、并发数(当前/上限)、本月拉取(已用/上限)、有效期剩余天数、状态颜色/图标、操作增加「查看详情」「延期」及一键复制授权码。

## 10.4 授权码延期

| 项 | 说明 | 状态 | 备注 |
|----|------|------|------|
| 选择授权码 → 延期 | 输入延期月数，更新 expires_at | ✅ | 后端 `PATCH /admin/licenses/{id}/extend`，body `{ "months": 1\|3\|6\|12 }` |
| 同步 registry_credentials.expires_at | 凭证有效期同步 | ✅ | 已实现 |
| 用户心跳自动感知 | 无需重新激活 | ✅ | 客户端按心跳返回的 expires_at 即可 |

**待办**：前端授权码列表/详情增加「延期」按钮，调用 `PATCH .../extend`，并需在 `admin` API 中增加 `licenseExtend(id, { months })` 方法。

## 10.5 数据总览

| 项 | 说明 | 状态 | 备注 |
|----|------|------|------|
| 各套餐存量分布（饼图） | 按套餐统计授权码数 | ⚠️ | 后端 `GET /admin/stats` 返回 `plan_breakdown`，**前端饼图目前是「授权码状态」used/unused/revoked**，非按套餐 |
| 本月新增激活数 | 本月激活的授权码数 | ⚠️ | 后端有 `new_activations_this_month`，**前端未展示**（仅展示「本月新增用户」） |
| 7 天内到期预警 | 点击查看名单 | ⚠️ | 后端有 `expiring_7_days`，前端仅展示「30 天内到期」列表，**未单独 7 天预警** |
| 各套餐活跃用户数 / 机器绑定总数 | 用户维度 | ⚠️ | 后端有 `active_users_by_plan`、`machine_bindings_total`，**前端未展示** |
| 本月拉取次数 Top 10 | 用户拉取排行 | ⚠️ | 后端有 `pull_top10_this_month`，**前端未展示** |
| 异常监控 | 拉取剩余≤3、并发满载、同机多账号 | ⚠️ | 后端有 `alerts`（同机多账号），**前端未展示**；拉取剩余≤3、并发满载需后端补逻辑或前端从详情聚合 |

**待办**：  
- 数据总览页：增加「各套餐存量」饼图（用 `plan_breakdown`）、「本月新增激活数」、单独「7 天内到期」区块（用 `expiring_7_days`）、「各套餐活跃用户数」「机器绑定总数」「本月拉取 Top 10」「异常监控」列表（用 `alerts` 等）。  
- 后端：`expiring_licenses` 需转为带 `plan_name` 等便于展示的 DTO（当前为 License ORM 列表，若直接序列化需保证字段一致）。
