# 数据互通机制 — 实现检查

对照「状态变更写库、三端查库感知」的数据互通机制，核对实现与文档的一致性。

## 4.1 状态变更的实时传播路径

### 作废授权码

文档要求：`PATCH /api/v1/admin/licenses/{id}/revoke` 后 FastAPI 执行 ①～④，三端通过查库/凭证立即感知。

| 步骤 | 文档描述 | 实现状态 | 说明 |
|------|----------|----------|------|
| ① | `license_keys.is_revoked = TRUE` | ✅ | `admin.revoke_license` 已设置 |
| ② | `registry_credentials.expires_at = NOW()`（立即过期） | ✅ | 作废时已更新该授权对应凭证的 expires_at |
| ③ | 从 htpasswd 文件删除该用户凭证 | ✅ | 作废时调用 `remove_from_htpasswd(registry_username)` |
| ④ | 写入 operation_logs | ✅ | 作废时写入 `OperationLog(action='revoke_license', target=license_id, ...)` |
| 路径 | 文档写 `.../licenses/{id}/revoke` | ⚠️ | 以实现为准：`PATCH /api/v1/admin/licenses/{license_id}`（无 body） |

**三端感知：**

| 端 | 文档描述 | 实现 |
|----|----------|------|
| 门户 | GET /user/license 返回「已作废」/ 授权失效 | ✅ 仅返回未作废授权，已作废用户视为未激活（is_activated=False） |
| Registry | 凭证已从 htpasswd 删除 → 401；或 Token 查 is_revoked/expires_at 拒绝 | ✅ proxy/token 校验 is_revoked；作废时已删 htpasswd 并置 expires_at |
| 心跳 | 查到 is_revoked → 返回 403，应用显示「授权已失效」 | ✅ 已作废授权返回 403，detail「授权已作废」 |

## 4.2 各管控限制的数据流

| 管控限制 | 数据存储位置 | 触发查询的时机 | 三端感知方式 | 实现 |
|---------|--------------|----------------|--------------|------|
| 授权有效期 | `license_keys.expires_at` | 每次 API 请求 | 门户剩余天数；Token 拒绝；心跳 403 | ✅ |
| 套餐模块权限 | `license_keys.module_mask` | 激活时 + Token/应用 | 门户模块入口；Registry Tag；应用 mask | ✅ 批量创建时写入 module_mask；激活时可按需补算 |
| 机器绑定数 | `machine_bindings` | 心跳上报时 | 门户列表；心跳超限 403；管理端查看 | ✅ |
| 并发实例数 | `instance_heartbeats` | 每次心跳 | 超限 403；管理端活跃数 | ✅ |
| 拉取次数 | `registry_credentials.pull_count_this_month` | 每次拉取 | 超限拒绝；门户剩余；管理端查看/重置 | ✅ |
| 生成次数 | `generation_quotas` | 新建项目/扣减时 | 超限 API 403；门户剩余；管理端查看/重置 | ✅ 按需创建 quota 行 |

## 4.3 授权码激活的完整联动流程

文档路径以实现为准：`POST /api/v1/user/activate`。

| 步骤 | 文档描述 | 实现 |
|------|----------|------|
| ① | 验证授权码格式（CRC 校验） | ✅ `validate_license_format` / LS 格式校验 |
| ② | 查 license_keys：存在 + 未使用 + 未过期 | ✅ `lookup_license_activate_status`（含 is_revoked） |
| ③ | 从 module_definitions 动态计算模块位图 | ⚠️ 当前在批量创建（v2）时写入 module_mask；激活接口可补算并写入 |
| ④ | 生成 Registry 凭证 username / password | ✅ `generate_registry_username` / `generate_registry_password` |
| ⑤ | 写入 htpasswd | ✅ `write_htpasswd` |
| ⑥ | AES 加密 password 写入 registry_credentials | ✅ `encrypt_password` + RegistryCredential |
| ⑦ | 写入 allowed_image_tags（根据套餐） | ✅ `get_allowed_image_tags(plan_code)` |
| ⑧ | 更新 license_keys：is_used, assigned_to, activated_at, expires_at | ✅ |
| ⑨ | 初始化 generation_quotas | ⚠️ 当前为首次扣减时按需创建（get_or_create_quota_row），非激活时预创建；行为等价 |

**结论**：数据互通机制已按文档实现；作废流程 ②③④ 与心跳 403 已补全；路径以《三系统职责边界》实现为准。激活步骤 ③、⑨ 为可选优化（module_mask 激活时补写、quota 预创建），不影响三端一致性。
