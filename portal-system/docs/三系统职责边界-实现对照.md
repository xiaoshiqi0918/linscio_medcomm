# 三系统职责边界

本文档以当前实现为准，API 路径与代码一致。

---

## 3.1 门户网站（用户侧）

**只做展示与触发，不持有任何业务状态。**

| 功能 | 调用的 API | 说明 |
|------|-------------|------|
| 注册/登录 | `POST /api/v1/auth/register` · `POST /api/v1/auth/login` | 获取用户 JWT |
| 激活授权码 | `POST /api/v1/user/activate` | 触发凭证生成 + htpasswd 写入 |
| 查看授权状态 | `GET /api/v1/user/license` | 实时从数据库读取 |
| 获取 Registry 凭证 | `GET /api/v1/registry/credential`（推荐）· 保留 `GET /api/v1/user/registry-credential` | 返回解密后的密码与 allowed_image_tags |
| 查看生成次数 | `GET /api/v1/user/quota-summary` | 返回各类型剩余次数 + 周期重置日 |
| 管理绑定机器 | `GET /api/v1/user/machines` · `DELETE /api/v1/user/machines/{binding_id}` | 查看/解绑机器 |

---

## 3.2 管理后台（运营侧）

**负责所有数据的写入与配置，变更立即影响其他两端。**

| 功能 | 调用的 API | 联动效果 |
|------|-------------|----------|
| 生成授权码 | `POST /api/v1/admin/licenses/batch` | 写入数据库，可分发激活 |
| 作废授权码 | `PATCH /api/v1/admin/licenses/{license_id}`（无 body） | 凭证立即失效，Registry 拒绝拉取，心跳返回 403 |
| 延期授权码 | `PATCH /api/v1/admin/licenses/{license_id}/extend`，body `{ "months": n }` | expires_at 更新，三端立即感知 |
| 配置模块权限 | `GET /api/v1/admin/modules` · `PATCH /api/v1/admin/modules/{id}` · `POST /api/v1/admin/modules` | 影响后续生成的授权码模块位图 |
| 批量更新凭证 | `POST /api/v1/admin/registry-credentials/sync-modules` | 已激活用户凭证同步新权限 |
| 单用户更新凭证 | `POST /api/v1/admin/users/{user_id}/refresh-credential` | 用户管理页对单用户刷新凭证（按当前套餐重算权限与密码） |
| 手动解绑机器 | `DELETE /api/v1/admin/licenses/{license_id}/machines/{binding_id}` | 立即释放绑定名额 |
| 重置生成次数 | `POST /api/v1/admin/quota/reset`，body `{ "license_id": "uuid", "machine_id": "可选" }` | 当前 90 天周期 used_count 置 0 |

## 3.3 Docker Registry（镜像仓库）

**只负责镜像存储与 Token 校验，不直接访问数据库。**

```
Registry 的所有权限判断均委托给 FastAPI Token 服务 / registry-proxy：

用户 docker login / 拉取
  → registry-proxy 或 FastAPI GET /api/v1/registry/token 查库验证凭证 + 套餐权限（含 is_revoked、allowed_image_tags、pull_limit_monthly）
  → 签发包含镜像访问范围的 JWT Token（有效期 1 小时），或 htpasswd + proxy 放行
  → Registry 凭校验结果放行请求

Token 过期后自动重新申请，无需用户感知。
套餐权限变更后，下次 Token 刷新 / 拉取时自动生效。
```

- 默认部署：htpasswd + registry-proxy（拉取前查库做 Tag 与拉取次数校验）。
- 可选：配置 Registry 使用 Token Auth，直接使用 `GET /api/v1/registry/token` 签发 JWT。
