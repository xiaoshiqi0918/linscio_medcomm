### 3.2 按套餐的镜像 Tag 管控（方案三）

| 项 | 状态 | 说明 |
|----|------|------|
| allowed_image_tags 按套餐写入 DB 并返回给用户 | ✅ | 激活时写入 registry_credentials.allowed_image_tags，接口返回 |
| 门户展示可拉取 Tag 示例 | ✅ | 见「门户」1.2 |
| Registry 拉取时校验「仅允许拉取 allowed_image_tags 内 Tag」 | ✅ | **已实现**：主应用提供 `app.registry_proxy` 代理服务；对 `GET /v2/<name>/manifests/<ref>`（ref 为 Tag 时）根据 Basic 用户名查 allowed_image_tags，不匹配则 403。docker-compose 中对外暴露 5000 的为 linscio-registry-proxy，转发到 linscio-registry。 |

**镜像仓库小结**：认证、凭证下发、**按 Tag 的拉取管控（registry-proxy）**均已实现。
