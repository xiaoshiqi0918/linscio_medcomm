# 门户系统数据库设计

本文档描述 portal-system 所用 PostgreSQL 表结构，与《LinScio_AI_分级授权与镜像管控方案_v5》及方案 SQL 一致。建表由 SQLAlchemy `Base.metadata.create_all` 执行，以下为规范 DDL 与预置数据，可作为手工建表或迁移依据。

**建表顺序**：先建 `portal_users`、`module_definitions`、`plans`、`billing_periods`，再建 `license_keys`、`registry_credentials`，最后建 `machine_bindings`、`instance_heartbeats`、`pull_records` 等。

---

## 8.1 数据表

```sql
-- 用户表
CREATE TABLE portal_users (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username      VARCHAR(50)  UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    email         VARCHAR(255),
    is_active     BOOLEAN      DEFAULT TRUE,
    created_at    TIMESTAMPTZ  DEFAULT NOW(),
    last_login    TIMESTAMPTZ
);

-- 授权码表（规范：无 plan_type，module_mask NOT NULL）
CREATE TABLE license_keys (
    id                  UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    code                VARCHAR(30)  UNIQUE NOT NULL,
    plan_id             UUID         REFERENCES plans(id),
    period_id           UUID         REFERENCES billing_periods(id),
    module_mask         INTEGER      NOT NULL,           -- 生成时根据模块配置计算
    machine_limit       INTEGER      NOT NULL DEFAULT 1,
    concurrent_limit    INTEGER      NOT NULL DEFAULT 1,
    pull_limit_monthly  INTEGER      DEFAULT NULL,
    created_by          VARCHAR(50)  NOT NULL,
    assigned_to         UUID         REFERENCES portal_users(id),
    activated_at        TIMESTAMPTZ,
    expires_at          TIMESTAMPTZ  NOT NULL,
    is_used             BOOLEAN      DEFAULT FALSE,
    is_revoked          BOOLEAN      DEFAULT FALSE,
    notes               TEXT,
    created_at          TIMESTAMPTZ  DEFAULT NOW()
);

-- Registry 凭证表
CREATE TABLE registry_credentials (
    id                    UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id               UUID         UNIQUE REFERENCES portal_users(id),
    license_id            UUID         REFERENCES license_keys(id),
    registry_username     VARCHAR(100) UNIQUE NOT NULL,
    registry_password_enc VARCHAR(500) NOT NULL,
    allowed_image_tags    TEXT[],
    pull_count_this_month INTEGER      DEFAULT 0,
    pull_count_reset_at   TIMESTAMPTZ,
    created_at            TIMESTAMPTZ  DEFAULT NOW(),
    expires_at            TIMESTAMPTZ
);

-- 管理员表
CREATE TABLE admin_users (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username      VARCHAR(50)  UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    scope         VARCHAR(20)  DEFAULT 'operator',     -- super / operator
    is_active     BOOLEAN      DEFAULT TRUE,
    created_at    TIMESTAMPTZ  DEFAULT NOW()
);

-- 操作日志表
CREATE TABLE operation_logs (
    id         BIGSERIAL    PRIMARY KEY,
    operator   VARCHAR(50),
    action     VARCHAR(100),
    target     VARCHAR(200),
    detail     JSONB,
    ip_addr    INET,
    created_at TIMESTAMPTZ  DEFAULT NOW()
);
```

**说明**：规范 SQL 中 `license_keys` 无 `plan_type`、`module_mask` 为 NOT NULL。当前 SQLAlchemy 模型为兼容旧码（LINSCIO- 等）保留 `plan_type`、`module_mask` 可 NULL，并增加 `replace_period_start`、`replace_count_used`（机器解绑周期次数，见 8.2）。新建环境若严格按规范建表，可先建 9.1 的 `module_definitions`、`plans`、`billing_periods`，再建本节的 `license_keys`、`registry_credentials`。

## 8.2 授权码格式

**旧格式（兼容）**：`LINSCIO-XXXX-XXXX-XXXX-XXXX`，字符集排除 O/0/I/1。实现：`license_service.py` 中 `LICENSE_CHARS`、`generate_license_code()`。

**新格式 v2（LS-）**：`LS-{套餐码}{周期码}-{时间哈希}-{随机段}-{校验位}`，示例 `LS-PY-A3K9-X7M2R4-Q8`；字符集排除 O/0/I/1/L；校验位为前段 CRC。实现：`generate_license_code_v2()`、`validate_license_format()`、`parse_license_code()`。

---

## 8.3 Registry 凭证生成逻辑

用户提交授权码激活时，后端执行：

1. **校验授权码**：格式校验（LS- 则 CRC 校验）；未使用、未作废、未过期。
2. **生成用户名**：新码 `{plan_prefix}_user_{uuid8}`（如 basic_user_a1b2c3d4），旧码 `user_`+uuid8。
3. **生成密码**：`secrets.token_hex(12)`（24 位十六进制）。
4. **写入 htpasswd**：同上。
5. **落库**：密码 AES 加密；`allowed_image_tags` 按套餐写入（新码）；`license_id` 关联授权码；`expires_at` = 授权码有效期。
6. **返回**：用户中心展示 Registry 地址、用户名、明文密码及可拉取镜像 Tag（`allowed_image_tags`）。

实现见 `api/app/routers/user.py`、`api/app/services/registry_service.py`。

## 9. 分级授权相关表（v2）

以下表与《LinScio_AI_分级授权与镜像管控方案_v5》第九章对应，用于 LS- 授权码、套餐/周期、镜像 Tag 管控。建表顺序：先建 `module_definitions`、`plans`、`billing_periods`，再建或迁移 `license_keys`、`registry_credentials`（含外键引用）。

### 9.1 模块定义、套餐、周期

```sql
CREATE TABLE module_definitions (
    id          UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    code        VARCHAR(50) UNIQUE NOT NULL,    -- 模块标识，如 schola / literature
    name        VARCHAR(100) NOT NULL,          -- 模块显示名称
    bit_position INTEGER     UNIQUE NOT NULL,   -- 在位图中的 bit 位（0~N）
    is_controlled BOOLEAN    DEFAULT TRUE,      -- TRUE=受控模块，FALSE=通用功能
    basic_enabled  BOOLEAN   DEFAULT FALSE,     -- 基础版是否开通
    pro_enabled    BOOLEAN   DEFAULT FALSE,     -- 专业版是否开通
    team_enabled   BOOLEAN   DEFAULT TRUE,      -- 团队版及以上默认全开
    sort_order     INTEGER   DEFAULT 0,
    created_at  TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE plans (
    id                  UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    code                VARCHAR(30)  UNIQUE NOT NULL,
    name                VARCHAR(60)  NOT NULL,
    plan_char           CHAR(1)      UNIQUE NOT NULL,
    machine_limit       INTEGER      NOT NULL DEFAULT 1,   -- 0 = 不限
    concurrent_limit    INTEGER      NOT NULL DEFAULT 1,   -- 0 = 不限
    pull_limit_monthly  INTEGER      DEFAULT NULL,        -- NULL = 不限
    price_monthly       NUMERIC(10,2) NOT NULL,
    is_active           BOOLEAN      DEFAULT TRUE
);

CREATE TABLE billing_periods (
    id            UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    code          VARCHAR(20)  UNIQUE NOT NULL,
    name          VARCHAR(60)  DEFAULT NULL,   -- 显示名（如 月付/年付），管理后台「套餐与周期」可配置
    period_char   CHAR(1)      UNIQUE NOT NULL,
    months        INTEGER      NOT NULL,
    discount_rate NUMERIC(3,2) NOT NULL DEFAULT 1.00
);
```

### 9.2 机器绑定、实例心跳、拉取记录

```sql
-- 机器绑定表
CREATE TABLE machine_bindings (
    id             UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    license_id     UUID         REFERENCES license_keys(id),
    user_id        UUID         REFERENCES portal_users(id),
    machine_id     VARCHAR(100) NOT NULL,
    machine_name   VARCHAR(200),
    first_seen     TIMESTAMPTZ  DEFAULT NOW(),
    last_heartbeat TIMESTAMPTZ  DEFAULT NOW(),
    is_active      BOOLEAN      DEFAULT TRUE,
    replace_count  INTEGER      DEFAULT 0,
    UNIQUE (license_id, machine_id)
);

-- 实例心跳表
CREATE TABLE instance_heartbeats (
    id           UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    license_id   UUID         REFERENCES license_keys(id),
    machine_id   VARCHAR(100) NOT NULL,
    instance_id  VARCHAR(100) NOT NULL,
    last_beat    TIMESTAMPTZ  DEFAULT NOW(),
    is_active    BOOLEAN      DEFAULT TRUE,
    UNIQUE (license_id, instance_id)
);

-- 拉取记录表
CREATE TABLE pull_records (
    id          BIGSERIAL    PRIMARY KEY,
    user_id     UUID         REFERENCES portal_users(id),
    license_id  UUID         REFERENCES license_keys(id),
    image_name  VARCHAR(200),
    machine_id  VARCHAR(100),
    is_update   BOOLEAN      DEFAULT TRUE,
    pulled_at   TIMESTAMPTZ  DEFAULT NOW()
);
```

主应用或 Registry 鉴权使用上述表；portal-system 已建表并预留 API。

### 9.3 生成次数管控（7.x）

每台机器每 90 天周期按内容类型（schola/medcomm/qcc）计数。表：generation_quotas（每机每周期每类型已用/上限）、generation_records（明细）。API：`POST /api/v1/quota/check` 校验并扣减；`GET /api/v1/quota/status` 查询本机本周期状态；`GET /admin/licenses/{id}/quota-usage` 管理端每机消耗。

## 8.4 数据备份策略

建议通过宝塔「计划任务」配置定时备份数据库与清理旧文件。

### 8.4.1 备份脚本（可选）

项目提供脚本 `scripts/backup-db.sh`，需在 **portal-system 目录下**执行（或保证该目录为当前目录），以便通过 `docker compose exec` 连接数据库容器。

**环境变量**（可选）：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `BACKUP_ROOT` | `/www/wwwroot/linscio/data/backups` | 备份文件目录 |
| `DB_CONTAINER` | `linscio-db` | Compose 中数据库服务名 |
| `POSTGRES_USER` | `linscio_user` | 数据库用户 |
| `POSTGRES_DB` | `linscio_portal` | 数据库名 |
| `RETENTION_DAYS` | `7` | 保留天数，超过的备份将被删除 |

**示例**：

```bash
cd /www/wwwroot/linscio/portal-system
./scripts/backup-db.sh
```

### 8.4.2 宝塔计划任务

- **任务类型**：Shell 脚本  
- **执行周期**：每天 3:00  
- **脚本内容**：

```bash
mkdir -p /www/wwwroot/linscio/data/backups
cd /www/wwwroot/linscio/portal-system && ./scripts/backup-db.sh
```

或直接使用一条命令（不依赖脚本）：

```bash
mkdir -p /www/wwwroot/linscio/data/backups
docker exec linscio-db pg_dump -U linscio_user linscio_portal \
  > /www/wwwroot/linscio/data/backups/portal_$(date +%Y%m%d_%H%M%S).sql
find /www/wwwroot/linscio/data/backups -name "*.sql" -mtime +7 -delete
```

若 Compose 容器名带项目前缀（如 `portal-system-linscio-db-1`），请将 `linscio-db` 改为实际容器名，或使用上述脚本（脚本通过 `docker compose exec` 按服务名执行，无需改容器名）。
