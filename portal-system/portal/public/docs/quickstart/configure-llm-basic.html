配置 LLM（首次必做） ​

部署完成后第一件重要事情：配置至少一个 LLM 服务商，以便各功能模块能调用大模型。

为什么必须配置 LLM ​

LinScio AI 的文献分析、论文写作、科普创作、数据分析、质控等模块均依赖大语言模型（LLM）生成与推理。未配置 LLM 时，相关功能将不可用或报错。

支持哪些 LLM 服务商 ​

  • DeepSeek（推荐新手）：国内直连，价格低，支持 deepseek-chat、deepseek-reasoner 等。
  • 阿里云 DashScope / 通义千问：国内直连，需阿里云账号与 API Key。
  • OpenAI（如 GPT-4）：需代理访问，配置 OPENAI_API_KEY 及可选代理。
  • Google Gemini：需代理访问，配置 GEMINI_API_KEY 及可选代理。
  • Ollama（本地）：无需 API Key，在服务器或本机安装 Ollama 并配置 OLLAMA_BASE_URL。

快速配置：以 DeepSeek 为例（推荐新手） ​

  1. 打开 DeepSeek 开放平台，注册/登录后进入 API 管理，创建 API Key 并复制。
  2. 登录 LinScio AI 系统，进入 系统设置 → LLM 与文献配置。
  3. 找到「DeepSeek API Key」，将复制的 Key 粘贴并保存。
  4. 在「智能体配置」页（可选）为各智能体选择模型，如 DeepSeek Chat 或 DeepSeek R1；不选则使用默认档位。
  5. 返回任意使用 LLM 的模块（如论文写作、文献分析），发起一次对话或分析，验证是否正常返回结果。

提示

若仅配置 DeepSeek，多数场景无需配置代理；若使用 OpenAI 或 Gemini，需在「代理配置」中填写 HTTP/HTTPS 代理地址。

配置完成后如何验证连接 ​

  • 方式一：在系统设置页保存 Key 后，若后端支持健康检查，可查看 LLM 服务状态是否变为可用。
  • 方式二：打开「论文写作」或「文献管理」等模块，输入一个问题或上传一篇文献并提问，能正常得到 AI 回复即表示连接成功。
  • 若出现 401/403，多为 API Key 错误或过期；若出现超时，请检查网络与代理配置。
