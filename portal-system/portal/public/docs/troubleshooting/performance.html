性能优化建议 ​

推荐 LLM Provider、并发与服务器配置、响应慢的排查路径。

推荐使用哪个 LLM Provider（延迟和价格对比） ​

  • DeepSeek：国内直连，延迟较低，价格友好，适合多数场景。
  • 通义/ DashScope：国内直连，模型选择多。
  • OpenAI / Gemini：需代理，延迟与稳定性受网络影响；适合对模型能力要求高的场景。
  • 具体价格与延迟以各厂商当前公布为准，可结合 LLM Provider 参数参考 选择。

并发用户数与服务器配置的关系 ​

  • 并发越高，对 CPU、内存与 LLM 调用 QPS 要求越高；建议根据实际并发数压测后调整服务器配置与 Provider 限流策略。
  • 数据库连接池、任务队列 worker 数等可参考项目部署文档调优。

响应慢的排查路径 ​

  1. 确认 LLM 请求是否耗时（查看网络延迟与模型响应时间）。
  2. 检查数据库是否有慢查询（日志或监控）。
  3. 检查任务队列是否积压；必要时增加 worker 或扩容。
  4. 若为代理访问海外 API，可尝试更换代理或改用国内 Provider。
