常见错误与解决方法 ​

启动时账号验证失败、无法连接门户、LLM 调用失败、模块不可用、Docker 容器启动失败等。

启动时「账号验证失败」 ​

主产品启动时会向门户校验账号或授权码，失败时可能报错并退出。根据返回的 error 或 detail 区分处理：

invalid_credentials：用户名/密码错误 ​

  • 原因：.env 中 LINSCIO_USERNAME 或 LINSCIO_PASSWORD 与门户账号不一致，或拼写错误。
  • 处理：检查 .env 配置，确认与门户登录账号一致；可在终端执行 grep LINSCIO .env 核对（注意不要泄露到公共环境）。修正后重启主产品。

not_purchased：未购买套餐 ​

  • 原因：该账号尚未购买或未激活授权，门户拒绝签发令牌。
  • 处理：在门户完成套餐购买或使用授权码激活；若为授权码路径，确认授权码已在用户中心激活且未作废。

account_suspended：账号已停用，需续费 ​

  • 原因：维护期已到期且超过宽限期，门户将账号置为 suspended，拒绝签发新令牌。
  • 处理：联系管理员或在门户办理年维护费续费，延长「维护截止日」后即可恢复；续费后重新启动主产品或等待自动续签。

machine_limit_exceeded：机器数超限，需解绑旧机器 ​

  • 原因：当前授权码已绑定机器数达到套餐上限，本机不在已绑定列表中。
  • 处理：登录门户用户中心，在「机器管理」或「已绑定机器」中解绑不再使用的机器，然后在本机重新启动主产品完成绑定。

启动时「无法连接到门户服务器」 ​

  • 原因：主产品无法访问 LINSCIO_PORTAL 配置的地址（如 https://api.linscio.com.cn），多为网络或防火墙问题。
  • 处理：
    1. 在部署机器上执行 curl -I <LINSCIO_PORTAL>/health 或浏览器访问门户健康检查地址，确认能否连通。
    2. 检查防火墙、安全组是否放行出站 HTTPS（443）。
    3. 若在内网，确认是否有代理或网关需配置；主产品需能解析并访问门户域名。

LLM 调用失败 ​

API Key 无效 ​

  • 返回 401/403 时，多为 API Key 错误、过期或未开通服务。
  • 处理：在系统设置中重新填写对应 Provider 的 API Key，或到服务商控制台检查 Key 状态与额度。

代理配置问题 ​

  • 国内访问 OpenAI/Gemini 等需代理，未配置或代理不可用时会出现连接超时或连接被拒绝。
  • 处理：在系统设置「代理配置」中填写正确的 HTTP/HTTPS 代理地址，保存后重试；或改用国内直连的 DeepSeek、通义。

服务商限速（429 错误） ​

  • 请求过于频繁触发服务商限流。
  • 处理：稍后重试；或配置多个 Provider 利用自动降级分散请求；必要时升级服务商套餐提高限流阈值。

某个功能模块无法使用 ​

套餐未包含该模块 ​

  • 当前套餐的模块权限不包含该功能，访问时返回 403 module_not_available。
  • 处理：升级套餐或联系管理员确认当前套餐包含的模块列表。

确认模块授权状态 ​

  • 在用户中心或「关于」页查看当前授权与模块列表；若应为可用但仍不可用，请检查主产品与门户版本是否匹配、是否有缓存，必要时重新登录或重启主产品。

Docker 容器启动失败 ​

端口被占用 ​

  • 报错 port is already allocated 等。
  • 处理：修改 docker-compose 中端口映射（如 8080 改为 8081），或停止占用端口的其他服务。

内存不足 ​

  • 容器或宿主机内存不足导致 OOM。
  • 处理：增加宿主机内存或为容器限制适当内存；关闭其他占用内存的服务。

镜像加载失败 ​

  • 本地无镜像且拉取失败（网络或仓库不可达）。
  • 处理：检查网络与镜像仓库地址；若使用安装包内的镜像 tar，先执行 docker load -i xxx.tar 再启动。
