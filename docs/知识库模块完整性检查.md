## 16.1 知识库 vs 文献支撑库

### 规范要求

| 维度 | 文献支撑库 | 知识库 |
|------|----------|-------|
| 来源 | 外部学术文献/指南 | 科室自建文档 |
| 格式 | PDF | PDF / TXT / Markdown |
| RAG 权重 | 0.6 | 0.4 |

### 当前状态

| 维度 | 文献支撑库 | 知识库 |
|------|----------|-------|
| 来源 | ✅ 文献支撑 | ✅ 科室知识 |
| 格式 | ✅ 仅 PDF | ✅ PDF / TXT / MD |
| RAG 权重 | ✅ 0.6 (paper_chunks) | ✅ 0.4 (knowledge_chunks) |

**结论**：16.1 已符合设计。

## 16.2 知识库 API

### 规范要求

```
POST   /api/v1/knowledge/docs/upload
GET    /api/v1/knowledge/docs
DELETE /api/v1/knowledge/docs/{id}
GET    /api/v1/knowledge/docs/{id}/status
POST   /api/v1/knowledge/docs/{id}/retry
```

### 当前状态

| 接口 | 路径 | 状态 | 说明 |
|------|------|------|------|
| POST upload | /api/v1/knowledge/docs/upload | ✅ | 上传 PDF/TXT/MD，后台索引 |
| GET list | /api/v1/knowledge/docs | ✅ | 文档列表 |
| DELETE | /api/v1/knowledge/docs/{id} | ✅ | 删除文档及 chunks、FTS 索引 |
| GET status | /api/v1/knowledge/docs/{id}/status | ✅ | 已实现 |
| POST retry | /api/v1/knowledge/docs/{id}/retry | ✅ | 已实现 |

### 缺口（已补齐）

1. ~~GET /docs/{id}/status~~ → 返回 `{ id, name, status }`
2. ~~POST /docs/{id}/retry~~ → 将 status 置为 indexing 并重新触发后台索引任务
