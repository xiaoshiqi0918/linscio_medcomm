## 一、规范要求概览

| 规范 | 要求 |
|------|------|
| **A.1** | 所有提示词使用版本号管理（PROMPT_VERSIONS），便于 A/B 测试和回溯 |
| **A.2** | 提示词优化流程：发现问题 → 定位 → 修改+版本+0.1 → 测试验证 → 对比评估 → 上线 → 记录 |
| **A.3** | Few-shot 示例数据质量：医生审核、风格达标、通过 AUTO_QUALITY_RULES、原创改写、来源标注 |
| **目录** | 创建 prompt-example 文件夹，所有提示词在其中，作为同步代码层的基础 |
