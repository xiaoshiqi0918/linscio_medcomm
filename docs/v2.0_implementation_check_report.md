## 二、部分实现 / 与文档存在差异 ⚠️

### 1. Layer 2 RAG 注入（v2.0 精炼摘要）

**文档要求**：
- 使用 RAG 过滤后的精炼摘要（相关性 ≥ 0.7 → top_insights ≤60字）
- `fallback_needed` 时输出「本次检索未找到高相关性参考资料……」
- `data_points` 中 `safe_to_use` 的可引用数据单独列出

**当前实现**：
- `prompt_builder._build_rag_section` 直接使用原始 `rag_chunks`，未接入 `rag_filter` 输出
- `auxiliary/rag_filter.txt` 和 `auxiliary_prompts.get_rag_filter_prompt` 已存在，但**未在 build_enhanced_prompt 流程中调用**

**建议**：在 RAG 检索后增加「调用 rag_filter 提示词 → 解析 JSON → 用 filtered_result 构建 rag_section」的流程。

---

### 2. Layer 2 章节上下文（context_section）

**文档要求**：
- v2.0 新增 `enhancement.context_section`（前序章节摘要 + 衔接指导）
- 组装顺序：`… enhancement.context_section, task_prompt …`

**当前实现**：
- `build_enhanced_prompt` 中无 `context_section` 参数和拼接逻辑
- `prompt_builder` 的 `parts` 仅含：rag_section, example_section, term_section, specialty_section, base_prompt

**建议**：增加 `context_section` 参数，并在有前序章节时注入衔接指导。

---

### 3. 重新生成改进提示（regenerate_hint）

**文档要求**：
- 重新生成时注入：`【本次重新生成的改进要求】\n{regenerate_hint}\n\n` + task_prompt

**当前实现**：
- `base.py` 和 `build_enhanced_prompt` 中无 `regenerate_hint` 处理
- 全库无 `regenerate_hint` 引用

**建议**：在 BaseAgent 调用增强流程时，若 `state.get('regenerate_hint')` 存在，则在 task_prompt 前追加该块。

---

### 4. SUGGEST_IMAGES_PROMPT 的 existing_image_anchors

**文档要求**：
- 输出格式中增加：`已有图像的段落前20字（跳过这些段落）：{existing_image_anchors}`

**当前实现**：
- `prompt-example/prompts/verification/suggest_images.txt` 无此字段
- `medcomm_nodes.py` 中 `SUGGEST_IMAGES_PROMPT.format()` 未传入 `existing_image_anchors`

**建议**：在 suggest_images 模板和调用处增加 `existing_image_anchors` 参数。

### 5. PlatformAdaptAgent 输出格式
