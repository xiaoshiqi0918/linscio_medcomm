## 四、缺口汇总

### P0（核心路径）

1. **多章节编辑**：当前只编首章，需支持章节切换、多 section 生成
2. **导出引擎**：按形式路由 HTML/DOCX/PDF/TXT，无实现
3. **防编造展示**：MedClaim、PendingClaim、ReadingLevel 等 Tiptap 节点
4. **Agent 群**：当前仅 ArticleIntroAgent，需覆盖 17 种形式的 section_type

### P1（重要模块）

5. **文献支撑库**：Paper 模型、上传 API、RAG paper_chunks
6. **知识库**：上传 API、解析、FTS5/向量索引
7. **图像生成**：DALL·E 3 接入、条漫逐格生成
8. **润色适配**：Polish API、Track Changes 前端
9. **模板库**：25+ 模板数据、API、新建时选择

### P2（增强）

10. **发布管理**：PublishRecord CRUD、阅读量录入 UI
11. **创作库**：与 MedComm 统一或独立视图
12. **设置页**：Ollama 状态、API Key 配置 UI、备份恢复
13. **快捷复制**：基于现有作品创建

## 五、建议开发优先级

```
Phase A（补齐核心路径）
├── 多章节编辑 + 多 section 生成
├── 导出引擎（至少 HTML/DOCX）
└── MedClaim/PendingClaim 节点 + 核实面板

Phase B（RAG + 数据）
├── 文献支撑库 API + 上传
├── 知识库 API + 上传
└── 模板库 API + 种子数据

Phase C（图像 + 润色）
├── 图像生成 DALL·E 3
├── 润色适配 + Track Changes
└── 发布管理
```
