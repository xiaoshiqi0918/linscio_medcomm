# Few-shot 示例模板完整性检查

基于规范 13.1–13.4 的示例模板，对比当前实现与规范要求的完整性。

---

## 一、模板规范要求概览

| 模板 | 形式 | 必备结构 | 分析要点 |
|------|------|----------|----------|
| 13.1 | 图文文章 | 示例类型 + 【高质量正文示例】 + 【分析】 | 类比、专业词解释、无绝对化、无编造、句长 |
| 13.2 | 口播脚本 | 示例类型 + 【高质量口播脚本示例】 + 【分析】 | 口语化、每句≤15字、停顿标记、节奏 |
| 13.3 | 条漫格 | 纯 JSON | panel_index, scene_desc(英文), dialogue, narration, caption, visual_notes |
| 13.4 | 患者手册 | 示例类型 + 【高质量示例】 + 【分析】 | 可操作性、情感支持、无具体用药 |

---

## 二、当前实现 vs 规范要求

### 2.1 图文文章示例（13.1）

| 规范要求 | 当前状态 | 缺口 |
|----------|----------|------|
| 示例类型：图文文章 / 正文章节 / 面向大众 / 微信公众号 / 内分泌科 | ❌ 无 | 种子无「示例类型」头部 |
| 【高质量正文示例】包装 | ❌ 无 | 种子仅纯正文 |
| 【分析】段落（✓ 类比、✓ 专业词解释、✓ 无绝对化…） | ❌ 无 | 无分析内容，LLM 无法学习质量标准 |
| 示例质量：有类比（燃料/油箱）、先正常后异常 | ⚠️ 部分 | 种子内容过于简略，无完整结构 |
| 句子平均 17 字、符合大众受众 | ⚠️ 未校验 | 未在种子或校验中体现 |

### 2.2 口播脚本示例（13.2）

| 规范要求 | 当前状态 | 缺口 |
|----------|----------|------|
| 示例类型：口播脚本 / 正文段 / 抖音 / 糖尿病主题 | ❌ 无 | 无头部 |
| [00:15-00:45] 医生（口播）： 时间戳格式 | ❌ 无 | 种子无时间戳 |
| 停顿标记（/） | ❌ 无 | 种子无停顿标记 |
| 【分析】✓ 完全口语化、✓ 每句≤15字、✓ 有反问… | ❌ 无 | 无分析 |
| 内容按字数换算约 90 字/30 秒 | ⚠️ 未体现 | 未在示例或说明中标注 |

### 2.3 条漫格示例（13.3）

| 规范要求 | 当前状态 | 缺口 |
|----------|----------|------|
| 输出为合法 JSON | ❌ 不符合 | 种子用 `content_text` 存纯文本，非 JSON |
| 字段：panel_index, scene_desc, dialogue, narration, caption, visual_notes | ❌ 不符合 | 种子格式为「[画面：…] narration：… dialogue：…」 |
| scene_desc 为英文 | ❌ 不符合 | 规范明确要求英文，种子可能含中文 |
| scene_desc 30–50 词 | ⚠️ 未约束 | 质量规则中有检测，示例本身未示范 |

### 2.4 患者手册示例（13.4）

| 规范要求 | 当前状态 | 缺口 |
|----------|----------|------|
| 示例类型：患者手册 / 日常注意事项 / 患者受众 / 糖尿病 | ❌ 无 | 无头部 |
| 结构化：**饮食管理**、**可以吃…**、**吃的顺序…** | ⚠️ 部分 | 种子为连续段落，无明确小标题结构 |
| 【分析】✓ 可操作、✓ 情感支持、✓ 未给用药建议 | ❌ 无 | 无分析 |
| 200g 等通俗单位需来源确认 | ⚠️ 规范已提醒 | 种子中有类似表述需审核 |

## 三、提示词注入模块（prompt_builder）

| 检查项 | 当前实现 | 缺口 |
|--------|----------|------|
| 示例头部 | `【示例 i】（platform · target_audience）` | 未用规范「示例类型：形式 / 章节 / 受众 / 平台 / 专科」 |
| 条漫格式 | 统一用 `content` / `content_text` | 条漫应优先用 `content_json`，并按 JSON 结构格式化输出 |
| 分析内容 | 不注入 | 【分析】未注入，LLM 无法参考质量标准 |
| 形式区分 | 所有形式同一模板 | 应按 content_format 分支：文章/口播/手册含分析，条漫用 JSON |

---

## 四、数据模型（WritingExample）

| 字段 | 用途 | 当前使用 |
|------|------|----------|
| content_text | 纯文本示例 | ✅ 文章/口播/手册使用 |
| content_json | JSON 示例（条漫等） | ❌ 种子条漫未使用，全存 content_text |
| — | 分析内容 | ❌ 无 `analysis_text` 或等效字段 |

**建议**：新增 `analysis_text` 字段，或约定将【分析】放在 `content_text` 末尾（需解析逻辑）。

---

## 五、示例类型中文映射（用于头部）

规范要求的「示例类型：图文文章 / 正文章节 / 面向大众 / 微信公众号 / 内分泌科」需要以下映射：

| 维度 | 枚举值 | 中文 |
|------|--------|------|
| content_format | article, oral_script, comic_strip, patient_handbook… | 图文文章、口播脚本、条漫、患者手册… |
| section_type | intro, body_1, panel_1, daily_care… | 正文章节、正文段、分格1、日常注意事项… |
| target_audience | public, patient | 面向大众、患者受众 |
| platform | wechat, douyin… | 微信公众号、抖音… |
| specialty | endocrine, cardiology… | 内分泌科、心内科… |

**当前**：`FORMAT_NAMES`、`SECTION_TITLES` 已存在；platform、audience、specialty 需补充中文映射。
