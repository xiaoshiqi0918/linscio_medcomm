# 打包与分发完整性检查

对照规范 22.1 安装包体积、22.2 启动时间。

---

## 22.1 安装包体积

| 组成 | 规范 | 当前状态 | 缺口 |
|------|------|----------|------|
| Electron | ~150MB | ✅ electron-builder 已配置 | 无 |
| python-build-standalone (3.11.12) | ~30MB | ❌ 使用系统 python3 | 未嵌入 |
| wheels（平台专属，jieba + WeasyPrint 等） | ~245MB | ❌ build/wheels/ 不存在 | 未预置 |
| assets/（图像提示词模板示例图 ~30 张） | ~15MB | ❌ 无 assets 目录 | 未准备 |
| 前端 + 后端编译产物 | ~20MB | ⚠️ dist + backend 已打包 | 需确认体积 |
| **合计** | **~460MB** | — | — |

### 当前实现

- **package.json build.files**：`electron/**`, `dist/**`, `backend/**`, `build/**`
- **backend.js**：`PYTHON_PATH || 'python3'`，依赖系统 Python
- **first-run.js**：期望 `build/wheels/<platform>/`，不存在则跳过
- **无 assets/**：未发现图像提示词模板示例图

## 22.2 启动时间

| 场景 | 规范 | 当前实现 | 缺口 |
|------|------|----------|------|
| 首次 | ≤ 90 秒 | 依赖 wheels 安装 + 迁移 | 无明确超时/目标 |
| 版本升级（迁移前备份 + 迁移） | ≤ 30 秒 | run_migrations.py 含备份 | 无显式计时/约束 |
| 正常启动 | ≤ 10 秒 | 轮询 /health 60s 超时 | 无启动耗时指标 |

### 启动时序（main.js）

1. localApiKey
2. Splash
3. Keychain
4. image-protocol
5. first-run wheels（无则跳过）
6. migration（含 Python 内备份）
7. 数据目录
8. spawn backend
9. pollHealth(60s)
10. 主窗口
