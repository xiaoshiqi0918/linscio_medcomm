# 提示词质量评估标准完整性检查

## 检查结论概览

| 模块 | 规范要求 | 当前实现 | 完整性 | 待办事项 |
|------|----------|----------|--------|----------|
| **QUALITY_CHECKLIST** | 分形式的质量关卡矩阵 | 无 | ❌ 未实现 | 新建 quality 模块 |
| **AUTO_QUALITY_RULES** | 正则检测规则（剂量/绝对化/超长句等） | 部分散落于 verification | ⚠️ 部分 | 集中实现 |
| **quality_check_node** | 接入自动化规则 | 仅检查过短、生成失败 | ⚠️ 简版 | 扩展检测项 |

---

## 12.1 输出质量评估矩阵

### 规范要求

| 形式族 | 质量关卡 |
|--------|----------|
| article_agents | 医学准确性、受众适配性、格式完整性、防编造合规、可读性 |
| script_agents | 口语化程度、格式合规、节奏合理、医学准确性 |
| comic_agents | JSON 合规、画面可生成性、对白精炼度、格间连贯性 |
| handbook_agents | 医嘱合规性、警示框准确、可操作性、PDF 格式兼容 |

### 当前实现

- **无 QUALITY_CHECKLIST** 定义
- quality_check_node 仅做：content_too_short、generation_failed
- verification pipeline 覆盖部分（剂量、绝对化），但未按形式族区分

---

## 12.2 质量检测自动化规则

### 规范要求

| 规则名 | 检测模式 | 级别 | 修复建议 |
|--------|----------|------|----------|
| 剂量表述 | 数字+单位 | error | 移除剂量，改为遵医嘱 |
| 绝对化表述 | 一定/必然/绝对等 | error | 替换为概率表述 |
| 编造来源 | 据…研究/根据…指南 | warning | 检查 RAG 支撑 |
| 超长句子 | 40+ 字 | warning | 拆分短句 |
| 英文缩写未解释 | 2–5 大写字母 | warning | 添加解释 |
| 感叹号密集 | 连续 3 个！ | warning | 减少感叹号 |
| JSON 格式 | 图示类 | check | 验证合法 JSON |
| scene_desc 中文 | 图示类 | error | 必须英文 |
| 画面描述太短 | scene_desc < 20 字符 | warning | 建议 30–50 词 |

### 当前实现

- **verification pipeline**：有剂量相关（FACT_VERIFY）、绝对化（_detect_absolute_terms），但为关键词列表，非完整正则
- **无 AUTO_QUALITY_RULES** 集中定义
- **无 scene_desc 专项检测**（图示类）

---

## 汇总：建议实施顺序

1. **P0**：新建 `app/services/quality/` 模块，定义 QUALITY_CHECKLIST、AUTO_QUALITY_RULES
2. **P0**：实现 `run_auto_quality_checks(content, content_format)`，按规则执行正则检测，返回 issues 列表
3. **P1**：quality_check_node 接入 run_auto_quality_checks，将 error 级问题纳入 regenerate 决策
4. **P2**：按 content_format 选用对应的 QUALITY_CHECKLIST 作为评估维度（供报告/展示）
